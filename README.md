## Adaptum BPMN (amoCRM widget)

Каркас виджета для amoCRM (под маркетплейс): UI в настройках + база для исполнения “процессов” по схемам (BPMN‑подобная модель).

### Что внутри

- `widget/` — фронтенд виджета (то, что загружается в amoCRM).
- `server/` — бэкенд (OAuth2, хранение токенов, API для виджета).

### Быстрый старт (локально)

1) Установи Node.js LTS (18+).

2) Запусти сервер:

```bash
cd server
npm i
npm run dev
```

3) Настрой `.env`:

Скопируй `server/.env.example` в `server/.env` и заполни значения.

### Важно про marketplace

Для публикации в маркетплейс нужно:

- зарегистрировать интеграцию/виджет в кабинете разработчика amoCRM
- получить `client_id`, `client_secret`
- настроить `redirect_uri` на публичный HTTPS домен
- отдать статические файлы `widget/` по HTTPS (обычно через тот же `server/`)

В этом репозитории всё подготовлено так, чтобы:

- виджет умел авторизоваться (OAuth) через бэкенд
- бэкенд был мульти‑аккаунтным (каждый аккаунт amoCRM хранится отдельно)
